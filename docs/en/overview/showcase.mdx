Locus is built for real Unity projects. Conversations, editor state, asset references, team knowledge, and Git review all live in the same desktop workspace, so the agent can carry a feature through an entire development chain.

## One Workspace, Full Loop

- Describe requirements and constraints in chat.
- Read project context: scripts, Prefabs, Scenes, Materials, ScriptableObjects.
- Drive the Unity plugin, asset index, and Git review from the same place.
- Pipe compile results, asset changes, and modification results back into the same conversation.

## Three High-Frequency Scenarios

### Fix a Real Feature

Read the relevant scripts and assets, make the changes, request a Unity recompile, and send the verification results straight back into the conversation.

### Investigate Asset Problems

Use the asset database, dependency graph, and risk report to track down `Missing Script`, broken references, duplicate GUIDs, and impact scope.

### Review Complex Changes

Put Git changes, file diff, and Unity semantic diff in one workspace so changes to Scenes, Prefabs, Materials, and similar assets are easier to read.
