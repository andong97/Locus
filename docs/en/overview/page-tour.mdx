## Session

![img](/images/overview/page-tour/session.png)

- You can input images in the message box (a multimodal model is required; GPT and Claude both work)
- Before using it, first make sure the asset database and Unity Editor in the upper left are working normally
- New conversation: click the `[+]` in the upper left or `[CTRL+N]` (the shortcut can be configured in Settings)
- The `[AUTO]` button in the lower right: in auto mode it will not ask for permission on every tool call. After switching it off, each modifying action will ask for permission. This is safer and more cumbersome
- The `[Model]` button in the lower right: select a model
- Send message: `[Enter]` by default. You can change it to `[CTRL+ENTER]` in Settings
- The session list on the left can be expanded by clicking the `[>]` on the left (to view subagents), and it supports the right-click menu (archive, delete) and multi-select while holding `SHIFT`

## Collaboration

![img](/images/overview/page-tour/collaboration.png)

- You can type Git commands or natural language directly into the command line at the bottom
- You can right-click commits and modified files to open some actions (the current set may still be incomplete; you can tell us what to add)
- Click the file list on the right (`commit`, `stash`, `unstaged/staged`) to view changes, including Unity YAML assets (`Scene`, `Prefab`, `ScriptableObject`, and so on)
- Field-by-field semantic merge is supported during conflicts
- Files marked as Locus are Locus knowledge/configuration files. If you do not want to commit them to Git, add them to `.gitignore`

## Knowledge

![img](/images/overview/page-tour/knowledge-types.png)

Knowledge is currently divided into four categories:

- `Design`: design requirement documents and fact sources. AI will automatically edit them after it receives requirements in a conversation (with your approval), and you can also write them yourself
- `Memory`: memory maintained automatically by AI. By default we provide three parts (user preferences, mistake notebook, project understanding). You can also create new documents and write maintenance rules, and AI will maintain them according to those rules
- `Skill`: the same kind of Agent Skill used by general agents. If you need the agent to execute the same process many times, you can write that process to the AI in natural language and let it follow the Skill every time
- `Reference`: external read-only documents. Right now you can import the Unity API Reference and Feishu docs

Documents support a **lexical index** (enabled by default) and a **semantic index** (enable it in the retrieval settings below)

> Lexical index: the AI searches document content using a few keywords
>
> Semantic index: the AI outputs a passage, and we search for document chunks with similar meaning
