---
title: vs. Generic Agents
description: Why Locus beats a generic coding agent inside Unity
---

Locus is more than a thin wrapper around an LLM API. We've put serious work into pushing Agent capability further.

- **JIT C# execution inside the Unity Editor.** With this, Locus modifies scenes and assets through scripts—no need to wrestle with giant YAML files on disk, and no need to pick from a sea of editor MCP tools.
- **Progressive-expansion intermediate representation for Unity assets and components.** Locus walks the hierarchy and loads only what it needs, instead of slurping and parsing whole files up front.
- **A full knowledge system built for game projects.** Auto-summarized design docs, long-term memory, Skill loading, and external imports (Unity API Reference, Feishu Docs, and more). Lexical and semantic retrieval ship side by side, with a local embedding runtime that runs out of the box on CPU or GPU.
- **A highly parallel Unity YAML semantic parser, written in Rust.** Locus reads many large files straight from disk semantically, and the same parser powers semantic diff and merge in version control.
- **A highly parallel Unity asset database, also in Rust.** Fast search across project assets and a complete bidirectional dependency graph, with direct queries for references and dependencies.

We're planning a technical blog post that walks through how Locus is built, along with the lessons we picked up along the way.
