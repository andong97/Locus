Looking at the current snapshot, Locus already has the skeleton of a full desktop workspace, the underlying runtime, and a fair amount of stability work in place.

## Maturity Signals at the Code Level

- The frontend is organized around six workspaces
- The Rust backend is split into independent modules: `agent / asset_db / diff / merge / knowledge_index / unity_bridge / llm / session / vcs`
- The Unity plugin has its own bridge implementation
- 93 test files currently live under `src/__tests__`
- The session persistence schema has reached `v10`, with explicit migration logic kept around

## Teams It Fits Best

- Build games in Unity
- Use Git for version control
- Want to actually embed AI into daily development
- Need to manage project knowledge, conventions, and external references in one place

## Current Product Focus

- Unity first
- Git first
- Local desktop workflow first
- Built around long-term project collaboration

## Implementation Basis

The claims on this page come from these modules in the current repository:

- App shell: `src/App.vue`
- Session and Agent UI: `src/components/ChatView.vue`, `src/components/AgentView.vue`
- Asset workspace: `src/components/AssetView.vue`, `src-tauri/src/asset_db/*`, `src-tauri/src/commands/asset.rs`
- Collaboration workspace: `src/components/CollabView.vue`, `src-tauri/src/merge/*`, `src-tauri/src/diff/*`
- Knowledge workspace: `src/components/KnowledgeView.vue`, `src-tauri/src/knowledge_index/*`, `src-tauri/src/knowledge_store.rs`
- Unity bridge: `src-tauri/src/unity_bridge/*`, `locus_unity/Editor/LocusBridge*.cs`
- Models and runtime: `src/stores/model.ts`, `src-tauri/src/llm/*`, `src-tauri/src/auth/*`
- Session persistence and migration: `src-tauri/src/session/store.rs`
