Locus organizes Unity assets as graph-level working objects and provides semantic diff and merge views that are closer to Unity structure for important resource types.

## Asset Database and Reference Graph

The current implementation already covers:

- Scanning `Assets` and `Packages`
- Parsing `.meta`, YAML assets, and script information
- Building asset nodes and reference edges
- Providing dependencies, reverse references, GUID resolution, and path resolution
- Outputting asset risk reports
- Detecting broken references, `Missing Script`, parse failures, and duplicate GUIDs
- Incrementally updating indexes through a filesystem watcher

This lets Locus answer many high-value questions in Unity projects:

- Which assets reference this Prefab
- Which resources are affected by a ScriptableObject change
- Where duplicate GUID risk exists in the current project
- Which assets already contain `Missing Script`

## Semantic Diff and Merge

The diff / merge modules in the repository already cover:

- Scene hierarchy and Inspector-structured views
- Semantic target construction for `PrefabInstance`
- Grouped field presentation for common object types such as materials and particle systems
- Dual views for semantic diff and text diff
- Merge semantic sessions and target materialization for Unity assets
- Conflict resolution workflow and merge panel

## Suitable Use Cases

- Asset troubleshooting and impact analysis
- Review of Scene, Prefab, Material, and ParticleSystem changes
- Conflict handling and merge assistance for Unity resources
