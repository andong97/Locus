---

Locus organizes its product capabilities around six workspaces. They share one goal: let the Agent work continuously inside the real context of a Unity project.

## Session

Daily conversation, task execution, tool-call confirmation, and context export all happen here. This is the Agent's main entry point into the project.

## Knowledge

Centralizes the four document types—`Design`, `Memory`, `Skill`, and `Reference`—and provides retrieval, injection preview, external imports, and maintenance entry points.

## Collaboration

Brings Git into the main workflow. Covers the history graph, staging area, conflict handling, semantic diff, and commit details. Useful when several people are touching Unity assets in parallel.

## Assets

Built around the Unity asset database and reference graph. Use it to inspect dependencies, reverse references, risk reports, and asset previews.

## Agent

Aggregates models, rules, system prompts, tool schemas, and runtime injection. The control surface for governing development Agents over the long run.

## Settings

Provider settings, model defaults, display preferences, shortcuts, knowledge configuration, runtime parameters—the foundation underneath everything else.

## Recommended Reading Path

Start with Session, Knowledge, and Collaboration. Once you start applying Locus to a real Unity project, dive into Assets and Agent.

- For the Unity execution loop: [Unity Editor Loop](/en/product/unity-editor-loop)
- For asset review and troubleshooting: [Asset Graph and Semantic Diff](/en/product/assets-and-diffs)
- For team accumulation and governance: [Knowledge System](/en/product/knowledge-system) and [Agent Governance](/en/product/agent-governance)
