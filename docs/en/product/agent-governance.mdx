Locus is not aimed at one-off Q&A. It is aimed at sustainable development agents. That is why it turns models, rules, tool confirmations, and runtime visualization into an independent governance layer.

## Current Governance Capabilities

- Multi-model access: OpenRouter, Claude subscription, Claude Agent SDK, OpenAI Codex, custom endpoints
- Plan mode and Build mode
- Sub-agent / task tools
- Tool execution confirmation and batch confirmation
- Undo tracking for file changes in each round
- Raw context export
- A visualization dashboard for the Agent system prompt, env, rules, runtime injection, and tool schema

## What These Capabilities Solve

- Let teams control the Agent's default behavior and boundaries
- Make tool calls and file changes reviewable and reversible
- Make model selection, rule injection, and runtime context more transparent in long-term projects

## Teams It Fits Better

- Teams that have already started using multiple models in parallel
- Teams that need to review Agent runtime behavior
- Teams that need to bring rules and knowledge into a stable governance process
