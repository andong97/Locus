Locus is built for sustained use, not one-off Q&A. That's why models, rules, tool confirmation, and runtime visibility are split out into a dedicated governance layer.

## Current Governance Capabilities

- Multi-model access: OpenRouter, Claude subscription, Claude Agent SDK, OpenAI Codex, custom endpoints
- Plan mode and Build mode
- Sub-agent / task tools
- Tool execution confirmation, with batch confirm
- Per-round undo tracking for file changes
- Raw context export
- A visual dashboard for the Agent's system prompt, env, rules, runtime injections, and tool schemas

## What This Solves

- Lets teams shape the Agent's default behavior and boundaries
- Keeps tool calls and file changes reviewable and reversible
- Makes model selection, rule injection, and context composition transparent over the long haul

## Teams It Fits Best

- Already running multiple models in parallel
- Need to inspect what the Agent is actually doing
- Want rules and knowledge to live in a stable governance flow
