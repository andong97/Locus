Locus links the file layer, editor layer, and compilation layer into one complete chain. This chain determines whether the Agent can really participate in actual Unity project development.

## Key Capabilities Already Covered

- Detecting Unity Editor connection state, including `editing / playing / playing_paused / disconnected`
- Selecting Unity assets from the desktop app and locating them in the editor
- Executing C# code on the Unity side
- Requesting recompilation after script changes and waiting for domain reload
- Importing changed assets in batches before recompilation
- Applying safety checks when Play Mode is active, the editor is disconnected, or state becomes inconsistent

## Why This Matters

Many real tasks are more than file edits. They usually include a whole execution chain:

1. Change scripts or assets.
2. Let Unity detect the changes correctly.
3. Wait for compilation or domain reload.
4. Return to the editor for verification.

Locus has already turned this chain into product capability, so it is closer to a real Unity development loop.

## Tasks It Is Suitable to Handle

- Request Unity recompilation immediately after script changes
- Trigger imports after batch-processing asset changes
- Sense state across runtime and edit mode
- Return error information, state changes, and verification results directly to the conversation
