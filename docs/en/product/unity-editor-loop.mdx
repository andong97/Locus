Locus stitches the file layer, the editor layer, and the compile layer into one chain. This chain decides whether the Agent can actually take part in Unity development, instead of just touching files.

## Capabilities Already Covered

- Detect Unity Editor connection state and distinguish `editing / playing / playing_paused / disconnected`
- Pick a Unity asset on the desktop side and locate it inside the Editor
- Execute C# code on the Unity side
- Request a recompile after script changes and wait for domain reload to finish
- Batch-import changed assets before recompile
- Run safety checks for Play Mode, disconnects, and inconsistent state

## Why the Chain Matters

Real tasks usually aren't single file edits. They tend to look like:

1. Change scripts or assets.
2. Get Unity to register the change.
3. Wait for the compile result or domain reload.
4. Return to the Editor to verify.

Locus has already made this loop a first-class product capability, so it sits much closer to a real Unity development cycle.

## Tasks It Handles Well

- Trigger a Unity recompile right after a script change
- Trigger an import after batch-processing asset changes
- Stay aware of state across runtime and edit mode
- Pipe errors, state changes, and verification results back into the session
