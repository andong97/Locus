Locus turns knowledge into an independent workspace and explicitly divides project knowledge into four categories, which makes long-term maintenance, unified retrieval, and stable injection easier.

## The Semantics of Four Knowledge Types

- `Design`: project design and direction
- `Memory`: long-term memory and user preferences
- `Skill`: standard workflows and SOP
- `Reference`: external materials

## Current Capabilities

- A unified knowledge tree and directory configuration
- Full-text retrieval and vector retrieval working in parallel
- Injection preview to inspect which knowledge blocks the current Agent will actually receive
- Local embedding runtime and remote embedding endpoint configuration
- Unity official documentation import
- Feishu knowledge base import
- External Reference directory integration

## Direct Value for Teams

This set of capabilities is suitable for bringing official docs, project standards, workflow skills, and project memory into one retrieval and injection system available to the Agent, reducing repeated context loss.
