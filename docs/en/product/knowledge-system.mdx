Locus turns knowledge into its own workspace and splits project knowledge into four categories, so it stays maintainable, searchable, and easy to inject reliably over time.

## The Four Knowledge Categories

- `Design`: project design and direction
- `Memory`: long-term memory and user preferences
- `Skill`: standard processes and SOPs
- `Reference`: external materials

## Current Capabilities

- Unified knowledge tree and directory configuration
- Full-text retrieval and vector retrieval running side by side
- Injection preview, so you can see exactly which knowledge blocks the Agent will receive
- Local embedding runtime, plus configurable remote embedding endpoints
- Unity official documentation import
- Feishu knowledge-base import
- External Reference directory integration

## Direct Value for Teams

This brings official docs, project conventions, Skill workflows, and project memory into a single searchable, injectable system the Agent can use—and cuts down on context that gets lost between sessions.
