Locus's direct value comes from three places: less context switching, better-quality Unity problem-solving, and a fit for long-term projects.

## Direct Value

### Less Context Switching

Unity, IDE, docs, wiki, and Git work all happen in one desktop workspace.

### Better Fit for Unity Project Problems

- Asset dependency and impact-scope analysis
- The Unity recompile loop after script changes
- Review of Scene / Prefab / Material / ParticleSystem changes
- Hunting down `Missing Script`, broken references, and duplicate GUIDs
- Pulling Unity's official docs and team knowledge into the same retrieval surface

### Better Fit for Long-Term Projects

Knowledge structure, session persistence, schema migration, context export, and undo tracking are all baked into the product as infrastructure.

## Typical Use Cases

### Fix a Real Feature in a Unity Project

- Read related scripts and Prefabs
- Check asset references
- Modify code
- Request a Unity recompile
- Send compiler errors or domain-reload state back into the session

### Troubleshoot Asset Problems

- Search for the target asset
- Review dependencies and reverse references
- Open the asset risk report
- Inspect `Missing Script`, broken references, and duplicate GUIDs

### Let the Agent Work With Project Knowledge

- Inject project design constraints
- Inject engineering conventions
- Inject common workflows
- Inject Unity's official docs and external knowledge bases

### Review and Merge Complex Asset Changes

When teammates touch Scenes, Prefabs, and Materials at the same time, Locus offers diff and merge views grounded in Unity semantics, so changes are easier to read.
