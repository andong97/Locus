The direct value of Locus comes from three aspects: less context switching, higher-quality handling of Unity problems, and better fit for long-term projects.

## Direct Value

### Less Context Switching

Developers can handle Unity, IDE, documentation, wiki, and Git-related work inside one desktop workspace.

### Better Fit for Unity Project Problem Types

- Analyzing asset dependencies and impact range
- Handling the Unity recompilation loop after script changes
- Reviewing Scene, Prefab, Material, and ParticleSystem changes
- Locating Missing Script issues, broken references, and duplicate GUIDs
- Bringing Unity official materials and team knowledge into the same retrieval entry point

### Better Fit for Long-Term Projects

Locus already turns knowledge structure, session persistence, schema migration, context export, and undo tracking into product infrastructure.

## Typical Use Cases

### Fix a Real Unity Feature

- Read the relevant scripts and Prefabs
- Check asset reference relationships
- Modify code
- Request Unity recompilation
- Return error results or domain reload state to the conversation

### Investigate Asset Problems

- Search for target assets
- Inspect dependencies and reverse references
- Open asset risk reports
- Check Missing Script issues, broken references, and duplicate GUIDs

### Let Agents Work With Project Knowledge

- Inject project design constraints
- Inject technical standards
- Inject common workflows
- Inject Unity documentation and external knowledge bases

### Review and Merge Complex Resource Changes

When team members modify Scene, Prefab, and Material at the same time, Locus can provide diff and merge views that are closer to Unity semantics and help developers understand changes faster.
