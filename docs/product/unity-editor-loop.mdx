Locus 把文件层、编辑器层和编译层串成了一条完整链路。这条链路决定了 Agent 是否能真正参与 Unity 项目的实际研发过程。

## 已覆盖的关键能力

- 检测 Unity 编辑器连接状态，区分 `editing / playing / playing_paused / disconnected`
- 从桌面端选中 Unity 资产并在编辑器里定位
- 在 Unity 侧执行 C# 代码
- 在脚本改动后请求重编译，并等待域重载完成
- 在重编译前批量导入变更资产
- 在 Play Mode、断连、状态不一致时做安全校验

## 这条链路为什么重要

很多真实任务都不是单纯地修改文件。它们通常包含一整条执行链：

1. 修改脚本或资源。
2. 让 Unity 正确感知这些变化。
3. 等待编译结果或域重载。
4. 回到编辑器继续验证。

Locus 已经把这条链路做成了产品能力，因此它更接近真实的 Unity 研发闭环。

## 适合承接的任务

- 修改脚本后立即请求 Unity 重编译
- 批量处理资源变更后触发导入
- 在运行态和编辑态之间做状态感知
- 把错误信息、状态变化和验证结果直接回传到会话
