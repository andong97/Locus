Locus 当前围绕六个主工作区组织产品能力。它们共同服务一个目标：让 Agent 能在 Unity 项目的真实上下文里连续工作。

## 会话

会话工作区承载日常对话、任务执行、工具调用确认和上下文导出。这里是 Agent 进入项目的主入口。

## 知识

知识工作区负责统一管理 `Design`、`Memory`、`Skill`、`Reference` 四类文档，并提供检索、注入预览、外部资料导入和维护入口。

## 协作

协作工作区把 Git 纳入主流程，覆盖历史图谱、暂存区、冲突处理、语义 diff 和提交详情，适合多人并行修改 Unity 项目资源时审查改动。

## 资产

资产工作区围绕 Unity 资源数据库和引用图展开，适合查看依赖、反向引用、风险报告和资源预览。

## Agent

Agent 工作区聚合模型、规则、系统提示词、工具 schema 和运行时注入信息，适合治理长期使用的开发代理。

## 设置

设置工作区负责 provider、模型默认值、显示偏好、快捷键、知识配置和运行参数，是整个桌面工作台的基础配置层。

## 推荐阅读路径

第一次了解 Locus 时，先看会话、知识、协作三个工作区；进入 Unity 项目落地时，再深入资产和 Agent。

- 关注 Unity 落地闭环：继续阅读 [Unity Editor 闭环](/product/unity-editor-loop)
- 关注资产审查与排障：继续阅读 [资产图谱与语义差异](/product/assets-and-diffs)
- 关注团队沉淀与治理：继续阅读 [知识系统](/product/knowledge-system) 和 [Agent 治理](/product/agent-governance)
