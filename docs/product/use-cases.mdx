Locus 的直接价值来自三个方面：减少上下文切换、提高 Unity 问题处理质量、适合长期项目持续使用。

## 直接价值

### 更少的上下文切换

开发者可以在一个统一的桌面工作台里完成 Unity、IDE、文档、Wiki 和 Git 相关操作。

### 更适合 Unity 项目的问题类型

- 分析资源依赖与影响范围
- 处理脚本改动后的 Unity 重编译闭环
- 审查 Scene / Prefab / Material / ParticleSystem 改动
- 定位 Missing Script、失效引用、重复 GUID
- 把 Unity 官方资料与团队知识纳入同一检索入口

### 更适合长期项目使用

Locus 已经把知识结构、会话持久化、schema 迁移、上下文导出和 undo 追踪做成了产品基础设施。

## 典型使用场景

### 修一个 Unity 项目里的真实功能

- 读取相关脚本和 Prefab
- 检查资产引用关系
- 修改代码
- 请求 Unity 重编译
- 把错误结果或域重载状态回传到会话

### 排查资源问题

- 搜索目标资源
- 查看依赖和反向引用
- 打开资产风险报告
- 检查 Missing Script、失效引用、重复 GUID

### 让 Agent 带着项目知识工作

- 注入项目设计约束
- 注入技术规范
- 注入常用工作流
- 注入 Unity 官方文档和外部知识库

### 审查和合并复杂资源改动

当团队成员同时修改 Scene、Prefab、Material 时，Locus 能提供更贴近 Unity 语义的 diff 与 merge 视图，帮助开发者更快看懂改动。
