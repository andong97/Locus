Locus 面向的不是一次性问答，而是可持续使用的开发代理。因此它把模型、规则、工具确认和运行时可视化做成了独立治理层。

## 当前治理能力

- 多模型接入：OpenRouter、Claude 订阅、Claude Agent SDK、OpenAI Codex、自定义端点
- Plan 模式与 Build 模式
- 子 agent / task 工具
- 工具执行确认与批量确认
- 每轮文件改动的 undo 跟踪
- 原始上下文导出
- Agent system prompt、env、规则、运行时注入、tool schema 的可视化仪表盘

## 这组能力解决什么问题

- 让团队能控制 Agent 的默认行为与边界
- 让工具调用和文件改动可审阅、可回退
- 让长期项目中的模型选择、规则注入和上下文构成更透明

## 更适合的团队状态

- 已经开始多模型并行使用
- 需要审查 Agent 运行过程
- 需要把规则和知识纳入稳定治理流程
