Locus 把知识做成了独立工作区，并把项目知识明确分成四类，方便长期维护、统一检索和稳定注入。

## 四类知识语义

- `Design`：项目设计与方向
- `Memory`：长期记忆与用户偏好
- `Skill`：标准流程与 SOP
- `Reference`：外部资料

## 当前能力

- 统一知识树与目录配置
- 全文检索与向量检索并行工作
- 注入预览，查看当前 Agent 实际会带入哪些知识块
- 本地 embedding 运行时与远端 embedding 端点配置
- Unity 官方文档导入
- 飞书知识库导入
- 外部 Reference 目录接入

## 对团队的直接价值

这套能力适合把官方文档、项目规范、技能流程和项目记忆统一纳入 Agent 可用的检索与注入体系，减少上下文反复丢失。

## 推荐和哪些能力一起看

- 与 [核心工作区](/product/workspaces) 一起看，理解知识在整个工作台里的位置
- 与 [Agent 治理](/product/agent-governance) 一起看，理解知识如何进入模型运行时
