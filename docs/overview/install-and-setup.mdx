## 配置模型服务

Locus必须配置一个远端的AI服务作为「大脑」，**远端大模型的能力会直接决定它的工作效果**。

> **Locus 目前支持的订阅服务以及消息格式**
> - OpenAI 订阅
> - Anthropic 订阅 （Anthropic已经明确不支持第三方客户端进行订阅，使用需要自行承担风险）
> - OpenAI Response API
> - Anthropic Messages API
> - OpenAI ChatCompeletions API

除了您配置的远端模型服务外，Locus是一个纯客户端软件，除模型调用外的任何信息都不会被发送到互联网上。

我们推荐您使用我们的安装包安装，安装成功后首次打开软件会显示引导流程：

## 选择项目并安装插件

![img](/images/overview/install-and-setup/unity-plugin-2.png)

Locus通过Unity插件与Unity编辑器通讯，以此实现检测编辑器状态、执行C#代码等功能。

插件默认会安装在Assets/Locus（后面可能会移动到Packages下），您可以手动在项目内移动插件的位置。

## 版本管理/Git配置

![img](/images/overview/install-and-setup/git-config.png)

Locus使用版本管理软件来实现「撤销」「变更分析」等功能。

另外我们对Git提供了图形化界面以供使用。

> 我们计划在未来支持P4、SVN更多版本管理软件，但目前仅支持Git

## 扫描资产数据库

![img](/images/overview/install-and-setup/asset-scan.png)

Locus不通过Unity编辑器来分析资产，相反它直接读取Unity在磁盘上的序列化文件以分析引用关系。

## 尝试第一次对话

<div style={{ textAlign: 'center', color: '#8f2f2f' }}>

Locus的配置部分已结束，您后续可以在「设置」分页中调整这些配置。

您可以尝试类似的对话：

- 帮我分析项目的代码结构，指出可以其中可以优化的地方？
- 我想要创建一个横版跳跃游戏，请编写一个简单的2D横版跳跃控制器，并在当前的场景中放置几个测试平台
- 帮我在Git上分批提交当前的修改
- 我想要创建自己的自定义渲染管线，应该是卡通风格的，制定一份方案供我审阅后实现
- 为项目中的所有'Item'资产下的描述字符串生成对应的多语言字符串填到表格中
