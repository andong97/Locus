---
title: 相对于通用 Agent
description: 相对于使用通用编程Agent，使用Locus的优势
---

Locus不仅是一个对于LLM API的简单封装，为了最大化Agent的能力，我们做出了很多努力。

- 我们实现了在Unity编辑器内JIT执行C#代码，Locus可以通过脚本语义化地修改Unity的场景与资产，而不是直接编辑在磁盘上的大型yaml文件，或者在上百个编辑器MCP工具间抉择要用哪个
- 我们对于场景、Prefab、组件按照「渐进式展开」的思想，设计了一套中间表示，Locus可以在层级结构内导航游览，按需读入，无需读入整个大型文件并自行解析结构
- 我们特定于游戏项目，实现了一整套知识系统，支持自动总结设计文档、维护记忆、加载skill、导入外部文档（例如Unity API Reference、飞书文档等）。支持词法/语义检索，原生支持本地嵌入向量运行时（支持本地CPU、GPU），无需复杂的配置，开箱即用。
- 我们基于Rust实现了一套高度并行化的Unity YAML语义解析器，Locus可以以此直接从磁盘上高速地语义化读入多个大型文件，并且支持在版本管理内进行语义化的diff & merge
- 我们基于Rust实现了一套高度并行化的Unity资产数据库，Locus可以以此高速搜索项目内的资产文件，并且维护了双向依赖关系，可以直接查找引用、依赖关系
