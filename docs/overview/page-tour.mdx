## 会话

![img](/images/overview/page-tour/session.png)

- 输入框内能够输入图片（必须使用多模态大模型，GPT/Claude都可以）
- 使用时，应该首先保证左上角的资产数据库、Unity编辑器正常运作
- 新建对话：点击左上角的【+】或者【CTRL+N】（在设置界面可以配置快捷键）
- 右下角的【AUTO】：自动模式下不会每次工具调用都向你请求权限，切换后每次进行修改操作都会请求权限，更安全、更麻烦
- 右下角的【Model】：可以选择模型
- 发送消息：默认【Enter】，可以在设置界面配置为【CTRL+ENTER】
- 左边的会话列表可以点左侧的【>】展开（查看subagent），并且支持右键菜单（归档、删除）与按住SHIFT的多选。

## 协作

![img](/images/overview/page-tour/collaboration.png)

- 你可以在下方的命令行内直接输入git指令或者自然语言
- commit、修改的文件上可以右键展开一些功能（可能目前配的不是很全，可以告诉我们新增功能）
- 点击右侧的文件列表（commit、stash、unstaged/staged）可以看到修改，包括Unity YAML资产（Scene、Prefab、ScriptableObject等）
- 冲突时支持逐字段语义合并
- 标记为Locus的文件是Locus的知识/配置文件，如果你不想要把它们提交到git中，可以把它们加入.gitignore

## 知识

![img](/images/overview/page-tour/knowledge-types.png)

知识目前分成四类：

- Design：设计需求文档与事实来源，AI在对话里收到需求后会自动编辑（需要你的审批），你也可以自己编写
- Memory：AI自动维护的记忆，默认我们设置了三块（用户偏好、错题本、项目理解），你也可以新建文档并编写维护规则，AI也会按规则自动维护
- Skill：与一般Agent Skill相同，如果你需要Agent多次执行一个同一个流程，你可以把这个流程用自然语言写给AI，每次按照Skill办事
- Reference：外部只读文档，目前你可以导入Unity API Reference和飞书文档

文档支持【词法索引】（默认开启）与【语义索引】（需要在下方的检索设置中打开）

> 词法索引：AI使用几个关键词查找文档中的内容
>
> 语义索引：AI输出一段话，我们检索有没有文档段落与这段话的意思相近
